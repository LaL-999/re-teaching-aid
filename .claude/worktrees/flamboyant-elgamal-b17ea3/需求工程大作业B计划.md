 一、公共功能

1. 用户登录
用户故事：作为教师或学生，我想使用学号和密码登录，以便系统识别我的角色并显示相应界面。
验收标准：
- Scenario: 教师登录成功
  Given 教师账号已创建，学号为“2024001”，密码为“123456”  
  When 教师在登录页输入学号“2024001”和密码“123456”，点击登录  
  Then 系统验证通过，跳转到教师主页  
  And 菜单显示“资源管理”、“作业管理”、“课堂演示工具”、“统计仪表盘”

- Scenario: 密码错误  
  Given 教师学号为“2024001”，正确密码为“123456”  
  When 教师输入密码“000000”并点击登录  
  Then 系统提示“学号或密码错误”  
  And 停留在登录页面

- Scenario3: 账号不存在 
  Given 学号“999999”未注册  
  When 用户输入学号“999999”和任意密码  
  Then 系统提示“账号不存在”

 二、教师端功能需求

2. 按模块（分为需求获取、需求建模、需求验证与确认、需求管理、需求优先级、需求规格说明书六个模块）上传课件及资源（支持ppt、txt、word、pdf文档）

用户故事：教师将课件及课程资源上传到指定教学模块，便于课堂按周次演示。

验收标准：

- Scenario: 成功上传课件 
  Given 教师已登录  
  When 教师选择模块“M1 需求工程概述”，填写资源名称“第1章 概述.pptx”，并上传合法的PPT文件  
  Then 系统保存资源，该课件出现在M1模块的资源列表中

- Scenario: 未选择模块 
  Given 教师已登录  
  When 教师填写资源名称和描述，上传文件，但未选择所属模块  
  Then 系统提示“请选择所属模块”

- Scenario: 上传不允许的文件类型 
  Given 教师已登录并选择了模块  
  When 教师上传一个.exe文件  
  Then 系统提示“不支持该文件类型”

3. 在线预览/展示资源

用户故事：教师在线预览课件或代码，用于课堂展示。

验收标准：

- Scenario: 预览PPT文件  
  Given 课件“第1章 概述.pptx”已上传  
  When 教师点击预览按钮  
  Then 系统在弹窗或新标签页中打开文件预览

- Scenario: 文件损坏无法预览 
  Given 课件文件已损坏  
  When 教师尝试预览  
  Then 系统提示“文件无法预览，请下载后打开”
4. 访谈提纲生成器（AI工具）

用户故事：教师输入项目领域，AI生成访谈提纲用于课堂演示。

验收标准：

- Scenario: 成功生成提纲 
  Given 教师已登录，打开AI工具“访谈提纲生成器”  
  When 教师输入“智慧图书馆”并点击生成  
  Then 系统调用大模型返回至少8个问题，支持一键复制到剪贴板

- Scenario: 输入为空 
  Given 教师未输入任何领域  
  When 点击生成  
  Then 系统提示“请输入项目领域”

- Scenario: API调用超时  
  Given 大模型服务响应超时  
  When 教师尝试生成  
  Then 系统提示“AI服务暂时不可用，请稍后重试”

5. i*模型代码生成（AI工具）

用户故事：教师输入需求描述，AI生成i*模型pistar代码。

验收标准：

- Scenario: 成功生成代码  
  Given 教师已登录，打开AI工具“i*模型代码生成”  
  When 教师输入“顾客希望准时送达，商家希望快速出餐，平台希望利润最大化”并点击生成  
  Then 系统返回代码，并提供复制和下载功能

- Scenario: 需求描述过于简单 
  Given 教师输入“配送快”  
  When 点击生成  
  Then 系统提示“需求描述过于简单，请补充更多细节”

- Scenario: API调用失败 
  Given 大模型服务异常  
  When 教师尝试生成  
  Then 系统提示“生成失败，请检查网络或稍后重试”

6. 需求质量检查器（AI工具）

用户故事：教师粘贴需求文本，AI检测模糊词和矛盾。

验收标准：

- Scenario: 检测出模糊词  
  Given 教师输入文本“系统响应要快，操作要方便”  
  When 点击“检查”  
  Then 系统标出“快”和“方便”为模糊词，建议改为具体指标

- Scenario: 文本为空  
  Given 教师未输入任何文本  
  When 点击“检查”  
  Then 系统提示“请输入需求文本”

7. UML模型代码生成与编译预览（AI工具）

用户故事：教师输入系统描述，AI生成UML用例图/活动图/类图的PlantUML代码，系统内编译预览图形。

验收标准：

- Scenario: 成功生成并编译预览  
  Given 教师已登录，打开AI工具“UML模型代码生成”  
  When 教师输入“外卖系统包括学生顾客、商家、骑手，学生可以提交订单、支付订单，商家可以接单，骑手可以配送”  
    And 选择图形类型为“用例图”  
    And 点击“生成代码”  
  Then 系统调用大模型返回PlantUML代码，并显示在代码编辑区  
    And 系统自动调用PlantUML引擎编译代码，显示生成的用例图预览  
    And 教师可复制代码或下载图片

- Scenario: 教师修改代码后重新编译 
  Given 系统已生成用例图PlantUML代码，预览显示正常  
  When 教师手动修改代码（例如添加一个参与者“支付系统”）  
  Then 系统自动重新编译并更新预览图形

- Scenario: AI生成无效代码导致编译失败  
  Given 教师输入模糊描述，大模型返回有语法错误的PlantUML代码  
  When 系统尝试编译  
  Then 编译失败，显示错误提示（例如“语法错误：缺少actor定义”）  
    And 教师可手动修改代码后重新编译

- Scenario: 未选择图形类型  
  Given 教师输入了描述但未选择图形类型  
  When 点击“生成代码”  
  Then 系统提示“请选择UML图形类型”

- Scenario: 需求描述过于简单  
  Given 教师输入“外卖系统”作为描述  
  When 点击生成  
  Then 系统提示“需求描述过于简单，请提供更详细的系统功能说明”

8. 需求规格说明书辅助生成（AI工具）

用户故事：教师输入项目背景和主要功能点，AI按GB/T 9385标准生成SRS初稿。

验收标准：

- Scenario: 成功生成SRS 
  Given 教师已登录，打开AI工具“需求规格说明书辅助生成”  
  When 教师输入项目名称“校园外卖系统”，主要功能点“用户注册登录、浏览商家、提交订单、支付订单、订单跟踪”  
    And 点击“生成SRS”  
  Then 系统调用大模型返回符合GB/T 9385结构的SRS文档（包含引言、总体描述、功能需求、非功能需求、接口需求等章节）  
    And 功能需求部分使用用户故事格式（As a… I want… so that…）  
    And 提供下载为Markdown或HTML的功能

- Scenario: 二次编辑后重新生成**  
  Given 系统已生成SRS初稿  
  When 教师修改其中的功能描述，点击“重新生成”  
  Then 系统基于修改后的输入重新生成SRS，保留非修改部分

- Scenario: 未输入项目名称或功能点为空
  Given 教师未填写项目名称，或功能点为空白  
  When 点击“生成SRS”  
  Then 系统提示“请填写项目名称和至少一个功能点”

- Scenario: AI生成内容缺失章节
  Given 教师输入完整信息，但大模型返回的SRS缺少非功能需求章节  
  When 教师查看生成的文档  
  Then 教师可手动补充，或点击“补充非功能需求”让AI重新生成缺失章节

三、学生端功能需求

9. 按模块浏览课件及资源

用户故事：学生按教学模块浏览教师上传的课件及资源。

验收标准：

- Scenario: 查看模块下课件列表  
  Given 学生已登录，模块M1下有课件“第1章 概述.pptx”  
  When 学生点击模块“M1 需求工程概述”

  When 学生点击模块“M2 需求获取——访谈”  
  Then 显示“暂无资源”

  Given 学生已登录

  Then 系统返回访谈问题列表，可复制
